# 🧩 1. 서비스 / 기능 / 상품 / 시스템 개요

## 🔷 프로젝트 개요

AI와 Azure 기반의 **간병인 매칭 및 관리 플랫폼**으로, 환자–간병인–가족–(확장 시) 간호사 간의

**일정 관리, 기록 추적, 보고 시스템**을 통합 제공하는 서비스입니다.

기존의 오프라인 중심 간병 시스템을 디지털화하여, **신뢰성 있는 간병인 매칭**과 **지속적인 케어 품질 관리**를 가능하게 합니다.

---

### 🔸 핵심 서비스 흐름

1. **회원 유형별 진입**
    - 환자/보호자 계정: 간병인 검색 및 예약
    - 간병인 계정: 프로필 등록, 근무 가능 일정 등록
    - 간호사 계정(확장): 전문 관리 및 평가 지원
2. **간병인 매칭 시스템**
    - 환자 상태(질환, 연령, 이동 보조 필요 등) + 근무 지역 + 성별/언어 등 선호 조건 기반 AI 추천
    - 리뷰, 평점, 근무 이력 기반 신뢰 점수 시스템
3. **일정 연동 및 관리**
    - 환자 일정과 간병인 스케줄 자동 동기화
    - Azure Calendar API 기반으로 알림 및 일정 변경 실시간 반영
4. **기록 및 리포트 시스템**
    - 간병인이 작성한 일일 케어 기록(식사, 복약, 운동, 상태 변화 등)
    - IoT 데이터(체온, 심박, 수면, 낙상 감지 등) 자동 기록 연동
    - 가족에게 주간/월간 리포트로 자동 전송
5. **확장 가능 구조**
    - 간호사/병원과의 연계로 환자 상태에 따른 맞춤 케어 관리
    - 장기적으로 의료보험 데이터와의 연계도 가능

---

### 🔸 차별화 포인트

- **IoT 연동 기록 추적**
    
    → 간병인 교체 시에도 데이터 손실 없이 환자 상태 추적 가능
    
- **AI 기반 간병인 추천 및 리포트 생성**
    
    → 케어 품질을 객관화하고, 가족에게 신뢰성 있는 리포트 제공
    
- **가족 중심의 투명한 케어 관리 시스템**
    
    → 가족이 실시간으로 환자 상태를 확인할 수 있는 모바일 알림 기능
    

---

### 🔸 데이터 전략

- **데이터셋 확보**
    - 공공 간병인/요양보호사 데이터셋 (예: 보건복지부, Kaggle, AI Hub 등)
    - 약 100명 규모의 간병인 프로필 데이터를 시범 구축 가능
- **내부 데이터 생성**
    - 서비스 이용 중 자동 수집되는 근무 기록, 평점, 건강 수치 등을
        
        정규화(normalization) 및 표준편차 기반으로 분석하여 학습 데이터 축적

## ⚙️ 2. 개발 / 기술 관점 (최소화된 핵심 구성)

### 🔹 AI / 데이터

- **Azure OpenAI Service**:
    - 리포트 요약, 환자 상태 분석, 추천 알고리즘 고도화
- **Azure Machine Learning**:
    - 매칭 알고리즘 모델 관리 및 훈련
- **데이터 표준화**:
    - Pandas/NumPy 기반 데이터 전처리 및 표준편차 기반 이상치 탐지

---

### 🔹 백엔드 / 인프라

- **플랫폼**: Microsoft Azure
- **백엔드 서비스**: Spring Boot / Python FastAPI 중 선택
- **데이터베이스**: Azure SQL Database 또는 Cosmos DB
- **IoT 연동**: Azure IoT Hub
    
    (체온계, 밴드, 낙상센서 등에서 실시간 데이터 수집)
    

---

### 🔹 프론트엔드 / 클라이언트

- **웹/앱 통합 플랫폼 (React + Tailwind / React Native)**
    - 보호자용: 리포트 및 스케줄 확인
    - 간병인용: 일정 등록 및 케어 기록 입력
    - 관리자용: 전체 매칭 및 데이터 모니터링

---

### 🔹 보안 및 인증

- Azure AD B2C를 통한 사용자 인증 (간병인/환자/가족)
- 개인정보 암호화 및 접근 권한 제어
- 로그 및 행동 분석을 통한 보안 감사(Audit)

---

### 🔹 향후 확장
