## ③ 핵심내용

### 🎯 문제 정의
- **기존 문제:** 간병인과 환자의 성향 불일치로 인한 낮은 만족도와 높은 이직률
- **해결 방안:** AI 성향 분석을 통한 **최적의 간병인-환자 매칭**

### 💡 핵심 기술
1. **PersonalityCalculator 모델**: 12개 성향 테스트 질문 → 4개 축 점수(공감도, 활동성, 인내심, 자립도) 자동 계산
2. **MatchingAlgorithm 모델**: 환자 성향 + 1,000명 간병사 데이터 → 호환성 점수 산출 → 상위 추천
3. **추천 엔진**: 6가지 호환성 지표(공감도, 활동성, 인내심, 자립도, 의료호환성, 성격일치) 종합 평가

## ④ 제안배경 및 목적

### 📋 배경
- **고령화 사회:** 2025년 기준 고령 인구 20% 이상, 가정간병 수요 급증
- **서비스 질 문제:** 성향 불일치로 인한 환자 만족도 저하 및 간병사 이직 심화
- **효율성 문제:** 수작업 매칭으로 인한 시간 낭비 및 부정확한 배치

### 🎯 목적
1. **환자 중심 매칭**: 성향 기반 최적 간병인 추천으로 서비스 만족도 ↑
2. **효율성 극대화**: 자동화된 AI 매칭으로 시간·비용 절감
3. **시스템 신뢰성**: 데이터 기반 의사결정으로 객관적 평가

## ⑤ 세부내용

### 📊 1. 활용 데이터 및 AI 모델

#### 데이터셋
- **Residents_korean.csv**: 1,000명 환자 정보 (이름, 나이, 성별, 입원일, 케어 레벨)
- **staff_korean.csv**: 1,000명 간병사 정보 (이름, 직급, 근무경력)
- **healthcare_dataset_korean.csv**: 의료 기록 및 진료 정보 (환자-간병사-진료 기록 연결)

#### AI 모델
- **PersonalityCalculator**: 성향 축별 점수 정규화 (0-100) 및 타입 분류
- **MatchingAlgorithm**: 다중 호환성 지표 계산 및 가중치 기반 점수 산출

### 🛠️ 2. 기술 구현

#### 백엔드 API
```
POST /api/personality/test
- 입력: 12개 성향 질문 답변
- 처리: PersonalityCalculator 모델 실행
- 출력: empathy, activity, patience, independence (0-100 점수)

GET /api/matching/recommend/{patient_id}
- 입력: 환자 ID
- 처리: MatchingAlgorithm으로 1,000명 간병사 처리
- 출력: 상위 10명 추천 (점수, 등급 A/B/C, 호환성 이유)
```

#### 프론트엔드 UI/UX
- **모바일 앱 기반**: 최대 너비 448px 제약 (반응형 모바일 우선 설계)
- **3단계 사용자 여정**:
  1. 🧪 성향 테스트: 12개 질문에 순차적 답변
  2. ⭐ 추천 결과: 간병사 카드 (점수, 호환성 지표, 추천 이유)
  3. 📊 매칭 현황: 활성/과거 매칭 관리

#### 데이터 흐름
```
프론트엔드 입력
  ↓
FastAPI 라우트
  ↓
비즈니스 로직(Service)
  ↓
AI 모델 실행
  ↓
SQLite 데이터베이스 저장
  ↓
JSON 응답
  ↓
React 컴포넌트 렌더링
```

### 🎨 3. UI/UX 주요 기능

| 페이지 | 기능 | 설명 |
|--------|------|------|
| **Welcome** | 서비스 소개 & 시작 | 간단한 설명과 "Get Started" 진입 |
| **PersonalityTest** | 성향 진단 | 12개 질문, 진행률 표시, 이전/다음/제출 버튼 |
| **Recommendations** | 맞춤 추천 | 간병사 카드 (점수, 등급, 호환성 지표, 추천 이유) |
| **Dashboard** | 매칭 관리 | 활성 매칭(녹색), 과거 매칭(회색), 취소 기능 |

### 🔄 4. 매칭 알고리즘 구조

```
환자 성향 점수 (4개 축)
    + 간병사 1,000명 스타일 데이터
    ↓
호환성 점수 계산:
  - empathy_match (공감도 차이 → 점수)
  - activity_match (활동성 차이 → 점수)
  - patience_match (인내심 차이 → 점수)
  - independence_match (자립도 차이 → 점수)
  - care_compatibility (의료 호환성 → 점수)
  - personality_compatibility (성격 일치도 → 점수)
    ↓
종합 매칭 점수 (0-100) 산출
    ↓
등급 부여: A (90+) / B (70~89) / C (50~69)
    ↓
상위 10명 정렬 및 추천
```

## ⑥ 기대효과

### 📈 사회·경제적 파급 효과

#### 1️⃣ **간병 서비스 품질 향상**
- 성향 기반 매칭으로 환자 만족도 ↑ (예상 30~40% 향상)
- 간병사-환자 관계 개선 → 서비스 질 체계적 관리

#### 2️⃣ **경제적 효율성**
- **매칭 시간 단축**: 수작업 1~2주 → 자동화 실시간
- **비용 절감**: 부정확한 배치로 인한 재배치 비용 감소
- **이직률 감소**: 적절한 매칭으로 간병사 이직률 ↓

#### 3️⃣ **사회적 가치**
- **고령층 삶의 질 향상**: 맞춤형 간병으로 심리적 안정감
- **간병 산업 기준화**: AI 기반 객관적 평가 제도화
- **일자리 창출**: 플랫폼 운영·유지보수 관련 고용 창출

#### 4️⃣ **확장성**
- 추후 **간병사/환자 관리 서비스** 연계
- 의료 기록 통합 → 통합 헬스케어 플랫폼 발전
- 시계열 데이터 축적 → 더욱 정확한 AI 모델 고도화
